# Documentation Generators Update Plan

## Overview
This plan outlines the update to the documentation generation system (`utl/doc`) to fully leverage the JSON output of the `ana` functions: `ana_laf`, `ana_acu`, and `ana_rdp`. These correspond to the alias blocks defined in `cfg/ali/sta` (`ffl-laf_cycle`, `ffl-acu_cycle`, `ffl-rdp_cycle`). 

The current implementations either drop valuable metadata or don't generate the appropriate Markdown files. The goal is to update these generators to read their scope from `utl/doc/config/targets`, invoke the `ana` functions using the `-j` flag (JSON output), parse that output, and format comprehensive Markdown reference documents in `doc/ref`.

## Tasks

### 1. Update `utl/doc/generators/func` (LAF Analysis)
- **Current State:** Parses JSON but discards most columns (Size, Loc, calls, etc.), extracting only `func_name` and `func_desc`.
- **Target File:** `doc/ref/functions.md`
- **Scope:** Reads from `TARGET_DIRECTORIES` in `utl/doc/config/targets` (specifically `LIB_CORE_DIR`, `LIB_OPS_DIR`, `LIB_GEN_DIR`, `UTL_DIR`).
- **Action:** 
  - Update the parsing logic (using `jq` if available, or robust shell loops/awk as done currently but expanded) to include all metadata extracted by `ana_laf -j` (Arguments, Shortname, Description, Size, Loc, etc.).
  - Rebuild the Markdown table in `doc/ref/functions.md` to reflect these comprehensive columns.

### 2. Update `utl/doc/generators/var` (ACU Analysis)
- **Current State:** Only outputs to standard out (stdout) via `ana_acu "-o"`. It does not write to the target documentation file.
- **Target File:** `doc/ref/variables.md`
- **Scope:** Variables defined in `CFG_CORE_DIR` and `CFG_ENV_DIR`, checked against usage in `LIB_DIR`, `SRC_DIR`, and `UTL_DIR`.
- **Action:**
  - Modify `utl/doc/generators/var` to run `ana_acu -j`.
  - Parse the generated JSON from `.tmp/doc/*.json`.
  - Format the variables, their values, total occurrences, and usage breakdown into Markdown tables, updating `doc/ref/variables.md` within designated auto-generated markers.

### 3. Create `utl/doc/generators/rdp` (RDP Analysis)
- **Current State:** No dedicated generator exists for reverse dependency tracking in documentation.
- **Target File:** `doc/ref/reverse-dependecies.md`
- **Scope:** Target dependencies across `LIB_CORE_DIR`, `LIB_GEN_DIR`, and `LIB_OPS_DIR` checked against modules in `DIR_BIN`, `LIB_DIR`, `SRC_DIR`, and `UTL_DIR`.
- **Action:**
  - Create the new script `utl/doc/generators/rdp`.
  - Run `ana_rdp -j` for the specified scopes.
  - Parse the `.tmp/doc/*.json` outputs to extract cross-module reverse dependencies.
  - Write or update `doc/ref/reverse-dependecies.md` with a structured list/table of function dependency calls.
  - Register the new generator and its target file in `utl/doc/config/targets` (and update `utl/doc/run_all_doc.sh` if necessary).

## Implementation Details
- Ensure `set +e` is correctly managed so that non-zero exits from grep/awk in the `ana` commands don't crash the generators.
- Use the `.tmp/doc/` path to retrieve the JSON generated by the `-j` options.
- Maintain consistent aesthetic "Brother in mind" layout for all three Markdown files in `doc/ref/`.
